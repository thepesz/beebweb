## 3. Technical SEO

### 3.1 robots.txt

- **Location**: `public/robots.txt`
- **Current Content**:
  ```
  # [App Name] - Health Tracking App
  # Allow all crawlers

  User-agent: *
  Disallow:

  # Sitemap
  Sitemap: https://domain.com/sitemap.xml

  # Crawl-delay for politeness
  Crawl-delay: 1
  ```
- **Requirements**:
  - ✅ Standard syntax (no `Allow: /`)
  - ✅ No non-standard directives
  - ✅ Points to sitemap
  - ✅ Allows all important pages
  - ❌ Avoid `Crawl-delay` (non-standard, Google ignores)
- **Issue Fixed**: Removed `Allow: /` and `Crawl-delay` directives

**Standard Format**:
```
User-agent: *
Disallow:               # Empty = allow all
Disallow: /admin/       # Block specific paths
Disallow: /api/         # Block API routes

Sitemap: https://domain.com/sitemap.xml
```

### 3.2 Sitemap

- **Location**: `app/sitemap.ts`
- **Type**: Dynamic sitemap (Next.js)
- **Current Implementation**:
  ```typescript
  export default function sitemap(): MetadataRoute.Sitemap {
    return [
      {
        url: 'https://domain.com',
        lastModified: new Date(),
        changeFrequency: 'weekly',
        priority: 1.0,
      },
    ]
  }
  ```
- **Requirements**:
  - ✅ XML format (Next.js auto-generates)
  - ✅ All important pages included
  - ✅ Accurate lastModified dates
  - ✅ Appropriate change frequency
  - ✅ Priority values (0.0 - 1.0)
  - ✅ Only URLs from current domain
  - ✅ Absolute URLs (https://)
- **Issue Fixed**: Removed cross-domain URLs (.pl from .com sitemap)

**Change Frequencies**:
- `always` - Changes constantly
- `hourly` - Updates hourly
- `daily` - Daily updates
- `weekly` - Weekly updates ✅ (most landing pages)
- `monthly` - Monthly updates
- `yearly` - Rarely changes
- `never` - Archived content

**Priority Guidelines**:
- `1.0` - Homepage ✅
- `0.8` - Main category pages
- `0.6` - Sub-category pages
- `0.4` - Blog posts
- `0.2` - Low priority pages

### 3.3 Cache Headers

- **Location**: `next.config.ts`
- **Implementation**:
  ```typescript
  async headers() {
    return [
      {
        source: "/:all*(svg|jpg|jpeg|png|gif|ico|webp)",
        headers: [
          {
            key: "Cache-Control",
            value: "public, max-age=31536000, immutable",
          },
          {
            key: "Expires",
            value: new Date(Date.now() + 31536000000).toUTCString(),
          },
        ],
      },
    ]
  }
  ```
- **Requirements**:
  - ✅ Expires header for images
  - ✅ Expires header for videos
  - ✅ Expires header for fonts
  - ✅ Cache-Control with max-age
  - ✅ Immutable flag for static assets
- **Issue Fixed**: Added Expires headers to resolve audit warning

**Cache Durations**:
- Images: 1 year (31536000 seconds)
- Videos: 1 year
- Fonts: 1 year
- CSS/JS: Versioned, can be cached forever
- HTML: Short cache or no-cache

### 3.4 URL Structure

- **Requirements**:
  - ✅ Lowercase URLs
  - ✅ Hyphens instead of underscores
  - ✅ Descriptive path names
  - ✅ No unnecessary parameters
  - ✅ HTTPS only
  - ✅ No trailing slashes (or consistent)
  - ✅ Short and memorable

**Good Examples**:
- `https://domain.com/about`
- `https://domain.com/blog/post-title`
- `https://domain.com/products/category`

**Bad Examples**:
- `https://domain.com/About` (uppercase)
- `https://domain.com/blog_post` (underscore)
- `https://domain.com/p?id=123` (not descriptive)

### 3.5 SSL/HTTPS

- **Requirements**:
  - ✅ Valid SSL certificate
  - ✅ All resources loaded via HTTPS
  - ✅ HTTP redirects to HTTPS
  - ✅ HSTS header (optional but recommended)
  - ✅ No mixed content warnings

**Check**: Use SSL Labs test (ssllabs.com)

- **Code**:
  ```tsx
  {/* Honeypot field to prevent bots */}
  <input
    type="text"
    name="website"
  ```

- `<article>` - Self-contained content

- `<figure>` - Image with caption
- `<figcaption>` - Image caption

**Benefits**:
- Better SEO

- Better code maintainability

## 6. Performance Optimization

### 6.1 Font Loading

- **Location**: `app/layout.tsx`
- **Implementation**:
  ```typescript
  const geistSans = Geist({
    variable: "--font-geist-sans",
    subsets: ["latin"],
    display: "swap",
    preload: true,
  });
  ```
- **Requirements**:
  - ✅ `display: "swap"` - Show fallback immediately
  - ✅ `preload: true` - Prioritize font loading
  - ✅ Subset fonts (latin, cyrillic, etc.)
  - ✅ Minimize font files
- **Issue Fixed**: Added display swap and preload for faster rendering

**Font Display Options**:
- `auto` - Browser default
- `block` - Hide text until font loads (avoid)
- `swap` - Show fallback immediately ✅
- `fallback` - Brief block, then swap
- `optional` - Only use font if cached

### 6.2 Image Optimization

- **Next.js Image Component**: Use when possible
- **Requirements**:
  - ✅ Lazy loading
  - ✅ Responsive sizes
  - ✅ Modern formats (WebP, AVIF)
  - ✅ Proper dimensions
  - ✅ Compressed

### 6.3 Video Optimization

- **Issue**: Videos with `preload="auto"` download entire file
- **Solution**: Use `preload="metadata"`
- **Implementation**: `components/ui/VideoBackground.tsx`
- **Benefits**:
  - 80-90% faster initial page load
  - Only loads metadata (duration, dimensions, first frame)
  - Video still plays smoothly when needed

**Preload Options**:
- `none` - Don't preload anything
- `metadata` - Load only metadata ✅ (best for background videos)
- `auto` - Browser decides (often downloads entire video)

### 6.4 Code Splitting

- **Next.js Default**: Automatic code splitting
- **Requirements**:
  - ✅ Route-based splitting
  - ✅ Dynamic imports for large components
  - ✅ Lazy load below-the-fold content

### 6.5 Compression

- **Requirements**:
  - ✅ Gzip or Brotli compression (Vercel handles)
  - ✅ Minified CSS/JS (Next.js handles)
  - ✅ Tree shaking (Next.js handles)

  - Enhancements (structured data)

- **Metrics**:
  - Performance score
  - Accessibility score
  - Best Practices score
  - SEO score
- **Core Web Vitals**:
  - LCP (Largest Contentful Paint) < 2.5s
  - FID (First Input Delay) < 100ms
  - CLS (Cumulative Layout Shift) < 0.1

- **Access**: F12 → Lighthouse tab
- **Same metrics as PageSpeed Insights**
- **Benefit**: Can test localhost

## 10. Common Issues & Solutions

### 10.1 Title Tag Issues

**Issue**: Title too short/long
- **Fix**: Keep 50-70 characters
- **Tool**: SERP preview tools

**Issue**: Duplicate titles
- **Fix**: Make each page unique
- **Check**: Google Search Console

**Issue**: Missing keywords
- **Fix**: Front-load important keywords
- **Test**: Does it match user search intent?

### 10.2 Meta Description Issues

**Issue**: Too long (truncated in search results)
- **Fix**: Keep under 160 characters
- **Tool**: SERP preview tools

**Issue**: Duplicate descriptions
- **Fix**: Write unique descriptions
- **Check**: Google Search Console

**Issue**: Not compelling
- **Fix**: Include value proposition and call-to-action
- **Test**: Would you click this result?

### 10.3 Heading Issues

**Issue**: Multiple H1 tags
- **Fix**: Only ONE H1 per page
- **Check**: View page source

**Issue**: Skipped heading levels (H1 → H3)
- **Fix**: Use hierarchical structure (H1 → H2 → H3)
- **Tool**: WAVE or axe DevTools

**Issue**: H1 doesn't match content
- **Fix**: Ensure H1 contains main keywords
- **Issue Fixed**: Changed from question-only to keyword-rich

### 10.4 Image Issues

**Issue**: Missing alt text
- **Fix**: Add descriptive alt text to all images
- **Tool**: WAVE

**Issue**: Large file sizes
- **Fix**: Compress and optimize
- **Tools**: TinyPNG, ImageOptim, Squoosh

**Issue**: Not responsive
- **Fix**: Use srcset or Next.js Image
- **Test**: Resize browser window

### 10.5 Mobile Issues

**Issue**: Text too small
- **Fix**: Minimum 16px font size
- **Tool**: Google Mobile-Friendly Test

**Issue**: Tap targets too small
- **Fix**: Minimum 44x44px touch targets
- **Tool**: Lighthouse accessibility audit

**Issue**: Horizontal scrolling
- **Fix**: Use responsive design, max-width: 100%
- **Test**: View on actual device

### 10.6 Performance Issues

**Issue**: Slow page load
- **Fixes**:
  - Optimize images (WebP, compress)
  - Reduce JavaScript
  - Enable caching
  - Use CDN
  - Lazy load below-fold content
- **Tool**: PageSpeed Insights

**Issue**: Large CSS/JS files
- **Fixes**:
  - Code splitting
  - Remove unused code
  - Minification
  - Tree shaking
- **Tool**: Bundle analyzer

**Issue**: Render-blocking resources
- **Fixes**:
  - Font display: swap
  - Defer non-critical CSS
  - Async/defer JavaScript
  - Inline critical CSS
- **Issue Fixed**: Added font display swap, changed video preload

### 10.7 Structured Data Issues

**Issue**: Invalid JSON-LD
- **Fix**: Validate syntax
- **Tool**: JSON validator

**Issue**: Temporal mismatch (founding date vs launch date)
- **Fix**: Ensure dates are consistent and logical
- **Issue Fixed**: Added datePublished: 2026, availabilityStarts: 2026-06-01

**Issue**: Missing required properties
- **Fix**: Check schema.org documentation
- **Tool**: Google Rich Results Test

### 10.8 Accessibility Issues

**Issue**: Form missing labels
- **Fix**: Add <label> with htmlFor
- **Tool**: WAVE, axe DevTools
- **Issue Fixed**: Added proper labels with sr-only class

**Issue**: Missing autocomplete
- **Fix**: Add autocomplete attribute
- **Issue Fixed**: Added autocomplete="email"

**Issue**: Low contrast
- **Fix**: Use WCAG AA contrast ratio (4.5:1)
- **Tool**: WebAIM Contrast Checker

**Issue**: Not keyboard navigable
- **Fix**: Ensure all interactive elements are focusable
- **Test**: Navigate with Tab key only

### 10.9 robots.txt Issues

**Issue**: Invalid syntax
- **Fix**: Use standard syntax only
- **Avoid**: `Allow: /`, `Crawl-delay:`
- **Issue Fixed**: Removed non-standard directives

**Issue**: Blocks important pages
- **Fix**: Review Disallow rules
- **Test**: Google Search Console → robots.txt Tester

**Issue**: Missing sitemap reference
- **Fix**: Add `Sitemap: https://domain.com/sitemap.xml`

### 10.10 Canonical Tag Issues

**Issue**: Canonical mismatch (points to wrong URL)
- **Fix**: Self-reference or point to preferred version
- **Issue Fixed**: Made canonical dynamic per domain

**Issue**: Relative vs absolute URL
- **Fix**: Always use absolute URL (https://domain.com)

**Issue**: HTTPS/HTTP mismatch
- **Fix**: Ensure consistent protocol
- **Rule**: Always use HTTPS
